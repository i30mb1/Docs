# Проброс портов при двойном NAT (ByFly + Xiaomi)

## Проблема

Статический IP на модеме ByFly, но за ним ещё роутер Xiaomi → двойной NAT → входящие соединения не доходят до компьютера.

## Решение

Модем ByFly перевести в режим моста — он просто пропускает трафик, а Xiaomi получает внешний IP напрямую.

## Шаги

### 1. Модем ByFly (192.168.1.1)

- Протокол: **Bridged**
- Режим моста: **Bridged PPPoE**
- Сохранить

### 2. Роутер Xiaomi (192.168.31.1)

![[Pasted image 20260311171737.png]]

- Тип подключения: **PPPoE**
- Логин/пароль: из договора ByFly
- Сохранить

### 3. Проброс портов (Xiaomi)

| Внешний порт | Внутренний IP | Внутренний порт | Протокол |
|--------------|---------------|-----------------|----------|
| 8081 | 192.168.x.x (IP компьютера) | 8090 | TCP |

## Проверка

1. [2ip.ru](https://2ip.ru) — должен показать ваш статический IP
2. [portchecker.co](https://portchecker.co) — порт должен быть открыт

## Схема "было → стало"

```
БЫЛО: Интернет → ByFly (NAT) → Xiaomi (NAT) → ПК  ❌
СТАЛО: Интернет → ByFly (мост) → Xiaomi (NAT) → ПК  ✓
```

## Как работает проброс портов

```
Внешний мир                             Локальная сеть
                                  ┌─────────────────────────────────┐
                                  │                                 │
Друг хочет зайти       Роутер     │  ┌─────────┐  Компьютер         │
на твой сайт           Xiaomi     │  │Телефон  │  с сервером        │
      │                  │        │  │ .1.11   │    .1.10           │
      │ 93.84.96.193:8081│        │  └─────────┘      │             │
      ├─────────────────►│        │                   │             │
      │                  │ проброс│  ┌─────────┐      │             │
      │                  ├────────┼──┼─────────┼─────►│ :8090       │
      │                  │  :8081 │  │Ноутбук  │      │             │
      │◄─────────────────┼────────┼──┼─────────┼──────┤             │
      │    ответ         │        │  │ .1.12   │                    │
                                  │  └─────────┘  Роутер знает:     │
                                  │               порт 8081 →       │
                                  │               комп .1.10:8090   │
                                  └─────────────────────────────────┘
```

**Зачем**: роутер имеет один внешний IP, но за ним много устройств. Проброс говорит роутеру: "всё что придёт на порт 8081 — отправляй на компьютер 192.168.1.10 порт 8090".
